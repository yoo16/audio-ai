# 04. 学習データの必要量・録音指針

GPT-SoVITS では **「音声の合計時間」が最重要**。ファイル数自体は本質ではない
(1 ファイルを 0b で分割すればセグメントは増えるため)。

## 合計の長さ

| 合計時間 | 品質 | 備考 |
|---|---|---|
| ~2分 (初回の試作) | 声質は似るが不安定 | 19 セグメント |
| **5〜10分** | **実用ライン (推奨)** | 安定性が大きく向上 |
| 15〜30分 | 高品質 | イントネーションも自然 |
| 30分〜1時間 | 最高品質 | これ以上は伸びが緩やか |

まず狙うのは **合計 5〜10分**。

## 1 セグメントの長さ

- 理想は **3〜10秒 / セグメント**
- 元ファイルは長くてよい (0b 音声分割が自動で切る)
- 1 秒未満や 15 秒超は避ける

## ファイル数の考え方

- 「合計 5〜10分」を満たせば数は問わない
- 例: 3〜5分の録音 2〜3 本でも、10秒の音声 50 本でもよい
- 分割後に **50〜100 セグメント**あると安定しやすい (初回は 19)

## 録音のコツ

- 一定のトーン・環境 (同じマイク・静かな部屋・BGM なし)
- ノイズや口ごもりの少ないクリアな音声
- 話し方に少しバリエーション (平叙・疑問など) があると表現力が増す

## データ追加時の再学習フロー

```
1. audio/ に音声を追加 (合計 5〜10分に)
2. python convert.py                 # audio_converted/ を再生成
3. WebUI: 0b 分割 → 0c ASR → 0d 校正
4. 1A 整形 (SV 抽出も忘れず) → SoVITS 学習 → GPT 学習
5. 推論で確認
```

一度パイプラインが通っていれば 2 周目は詰まらない。
