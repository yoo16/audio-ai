# 03. 推論と品質チューニング

## 推論 WebUI 起動

GPT-SoVITS TTS タブの「1C-推論」→「推論WebUIを開く」で、別ポート **http://localhost:9872** に推論画面が立ち上がる。

## 基本設定

1. **GPTモデル(ckpt)**: `yoo-e15.ckpt`
2. **SoVITSモデル(pth)**: `yoo_e8_s200.pth`
   - ※ デフォルトはベースモデルなので、必ず自分の `yoo-*` に変更する
3. **参照音声** (声色の見本, 3〜10秒): `output/slicer_opt/` から明瞭なものを 1 つ
4. **参照音声のテキスト**: その音声の実際の発話内容 (正確に入れる)
5. **参照音声の言語**: 日本語
6. **合成したいテキスト** + **言語**: 日本語
7. 「合成開始」

参照音声の例:
```
output/slicer_opt/login_1.wav_0000021760_0000177280.wav
テキスト: 第12回第1項ログイン処理その1について講義します
```

## 品質チューニング (文章が不安定なとき)

少量データ (〜2分) では不安定になりやすい。以下で改善する。

### パラメータ (効果大)

| パラメータ | デフォルト | 安定化 | 効果 |
|---|---|---|---|
| top_k | 15 | **5〜10** | ランダム性を下げる |
| top_p | 1.0 | **0.8〜0.9** | 同上 |
| temperature | 1.0 | **0.6〜0.8** | 暴れを抑える |
| repetition_penalty | 1.35 | 1.35 | 繰り返し防止 |

→ まず **temperature を 0.7、top_k を 8** に下げるのが最も効く。

### テキスト分割方法 (効果大)

「how to cut / テキストの分割方法」を **「句読点で分割」** など細かめに。
長文を一度に生成すると不安定になるため、短く区切る。

### 参照音声

- 3〜6秒で、明瞭・平坦なトーンのもの
- 参照テキストが音声と完全一致しているか確認
- 別の参照音声も試す (`hikaku_2`, `browser_1` など)

### 生成テキスト

- 極端に長い文を避け、適度な句読点を入れる
- 学習データに近い口調 (今回は講義口調) だと安定

## 根本的な改善

パラメータ調整で足りない場合は、**データを増やして再学習**が最も効果的。
必要量は [04-dataset.md](04-dataset.md) を参照。
